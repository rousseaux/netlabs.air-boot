 º                                                                            º

 º VIRUS DETECTION ³                                                          º
 ºÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÙ                                                          º
 º AiR-BOOT includes 3 different methods of virii detection.                  º
 º Almost any boot-virus should be covered with those.                        º
 º                                                                            º
 º Normal - This one will compare the MBR in memory with the MBR on drive at  º
 º           each boot. So normal MBR virii will get found.                   º
 º                                                                            º
 º Stealth - This one will detect virii that hide themselves and fake BIOS    º
 º            HDD access API. Normal detection would fail on those virii.     º
 º                                                                            º
 º VIBR - Virus-In-Boot-Record Detection. Requires the global switch AND the  º
 º         specific partition-flag to be enabled. This one will check the     º
 º         bootrecord of the selected partition for modifications.            º
 º         This detection is NASTY and may report false positives.            º
 º         If a modified boot-record is found, booting will be denied from    º
 º         that partition. You may reset this check by toggling the V-flag.   º
 º                                                                            º

 º It will capture INT 13h (Low level I/O) and watch for writes to cylinder 0 º
 º and head 0. This will be regarded as a viral act and aborted. The computer º
 º will stop and you will see a corresponding message. This option could be   º
 º incompatible to a 3rd company program, like the option 'Virus Protection'  º

 º EXTENDED PARTITION MS WORK-AROUND ³                                        º
 ºÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÄÙ                                        º
 º This 'feature' needed to be implemented, because of Microsofts way of      º
 º inventing a pseudo-standard that is incompatible with any other OS.        º
 º                                                                            º
 º The problem is with the id of extended partitions. The official standard   º
 º was and is to use '05h'. Now Microsoft "invented" a new id of '0Fh' for    º
 º extended partitions that occupy space above 8 GB.                          º
 º                                                                            º
 º But now many OS won't see ANY logical partitions anymore. So a smart user  º
 º could try to change it back. This works, but (of course) some MS programs  º

 º partitions.                                                                º
 º                                                                            º
 º So we got the actual standard, that will work somehow even with Windows,   º
 º but sometimes partitions may (!) get destroyed.                            º
 º Or we use the MS invention, but lose compatibility to some OSes.           º
 º ...or we use AiR-BOOT to switch between those 2 "standards" on-the-fly.    º
 º                                                                            º
 º When the global switch AND the P-flag on the booted partition is enabled,  º
 º AiR-BOOT will change all extended partitions to be of id '0Fh', so the     º
 º system will be compatible to MS Windows.                                   º
 º                                                                            º
 º On the other hand, if the global switch is enabled, but the P-flag is      º
 º disabled, AiR-BOOT will change them to the real standard, which means for  º
 º example DOS will recognize the logical partitions again.                   º
